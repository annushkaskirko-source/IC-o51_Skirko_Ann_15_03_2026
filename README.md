## Мета роботи
Реалізувати правила предметної області та перевірку коректності введених даних.

## Завдання роботи
1. Реалізувати основні бізнес-правила предметної області.  
2. Реалізувати перевірку коректності введених даних.  
3. Використати атрибути валідації.  
4. Реалізувати повідомлення про помилки.  

## Використані технології та підходи
- **Мова програмування:** C#  
- **Валідація даних:** Атрибути `[Required]`, `[Range]`, `[StringLength]`  
- **Обробка винятків:** try-catch для помилкових введень  
- **Бізнес-логіка:** Перевірка правил предметної області перед збереженням об’єктів  

## Опис реалізованих бізнес-правил
- Всі об’єкти повинні мати унікальні імена.  
- Значення рівня персонажа повинно бути в межах 1–100.  
- Обов’язкові поля не можуть залишатися пустими.
